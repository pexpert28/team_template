# Team Onboarding

Welcome to the team! This page gets you from zero to productive as fast as possible.

---

## Step 1 — Access

- [ ] Get added to the GitHub org by your team lead
- [ ] Get added to the `@YOUR_ORG/YOUR_TEAM_NAME` GitHub Team
- [ ] Clone the repo and follow the setup steps in [CONTRIBUTING.md](../CONTRIBUTING.md)
- [ ] Get access to any required services (see your team lead for credentials/secrets)

## Step 3 — First contribution

A good first PR is usually a docs fix, a small test, or a config tweak.  
Check the issues labelled `good first issue` for something to pick up.

---

## Team norms

- We use **GitHub Issues** for tracking work — don't start work without an issue
- PRs should be reviewed within **1 business day**
- Discussions about design/architecture go in **Team Discussions** (GitHub Teams tab)
- Breaking changes get a heads-up in the team discussion before a PR is opened
